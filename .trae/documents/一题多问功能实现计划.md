# 一题多问功能实现计划

## 功能概述
为错题系统添加"一题多问"功能，应对数学大题或语文阅读题等一个材料多个问的场景。

## 核心需求
1. **增加小问功能**：在原题区域增加类似卡片的小问编辑区
2. **小问内容**：每个小问包含题目原文（富文本）和标签
3. **标签继承**：小问继承大题的标签（仅用于搜索，保存时只保存小问自己的标签）
4. **预览/打印**：小问内容不显示在预览和打印中（仅作为数据标记）

---

## 实现步骤

### 1. 数据结构设计

**JSON保存格式（`index.html` 和 `app.py`）**：
```javascript
{
  "id": "xxx",
  "question": { "items": [...] },
  "answer": { "items": [...] },
  "tags": ["大题标签1", "大题标签2"],
  "sub_questions": [
    {
      "id": 1,
      "question_text": { "items": [...] },  // 小问题目原文
      "tags": ["小问标签"]  // 小问自己的标签
    },
    {
      "id": 2,
      "question_text": { "items": [...] },
      "tags": []
    }
  ]
}
```

- `/api/save` - 保存时包含 `sub_questions`
- `/api/questions` - 返回时包含 `sub_questions`
- `/api/questions/<id>` PUT - 更新时处理 `sub_questions`

## 关键实现细节

### 小问标签继承逻辑（搜索用）
```javascript
// 获取题目的所有有效标签（包含继承的标签）
function getEffectiveTags(question) {
  const ownTags = question.tags || [];
  const subQuestionTags = (question.sub_questions || [])
    .flatMap(sq => sq.tags || []);
  return [...new Set([...ownTags, ...subQuestionTags])];
}
```

### 搜索时使用有效标签
修改 `browse.html` 中的 `evaluateAST` 函数，使用 `getEffectiveTags` 获取题目的所有标签进行匹配。

### 小问卡片的编辑功能
在 `browse.html` 的编辑视图中，可以：
- 查看所有小问
- 编辑小问的标签
- 删除小问
- 添加新小问

## 标签继承说明

- **保存时**：小问只保存自己的 `tags`
- **搜索时**：使用 `getEffectiveTags` 函数，将父题目标签与小问标签合并后匹配
- **显示时**：编辑区显示小问自己的标签

这样设计确保：
1. 每道题的标签数据独立存储
2. 搜索时可以匹配到父题目的标签
3. 预览打印不显示小问内容（纯数据标记）
